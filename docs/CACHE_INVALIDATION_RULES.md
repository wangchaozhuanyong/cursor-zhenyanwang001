# 缓存和失效规则

适用范围：前台公开数据、后台管理数据、PWA/浏览器缓存、服务端内存/Redis 缓存。

## 1. 总原则

1. 私密数据、后台接口、登录态、订单、购物车、支付、上传接口：不允许公共缓存。
2. 公开且高频读取的数据可以缓存，但必须有过期时间。
3. 后台修改会影响前台展示时，保存成功后必须清理对应缓存，或者把缓存 TTL 控制在可接受范围内。
4. 任何新增缓存都要同时写清楚：缓存什么、缓存多久、谁负责失效、失败时怎么办。

## 2. 现有缓存位置

| 缓存位置 | 文件 | 当前规则 |
| --- | --- | --- |
| 服务端通用 Redis 缓存 | `server/src/utils/cache.js` | 默认 TTL 300 秒，可通过 `REDIS_CACHE_TTL_SECONDS` 调整。 |
| 商品/分类/轮播服务端内存缓存 | `server/src/modules/product/service/catalog.service.js` | TTL 60 秒，后台商品、分类、活动改动后调用 `clearCatalogCache()`。 |
| 前端 React Query | `click-send-shop-main/click-send-shop-main/src/lib/queryClient.ts` | 默认 `staleTime` 5 分钟；后台关键页一般单独设置 30-60 秒并在保存后 invalidate。 |
| PWA 运行时缓存 | `click-send-shop-main/click-send-shop-main/vite.config.ts` | 后台、登录、订单、购物车、支付、上传为 `NetworkOnly`；公开数据按类型设置 1 分钟到 24 小时。 |
| 静态资源缓存 | `server/src/app.js` | 构建产物 `/assets` 缓存 30 天；上传图片 `/uploads` 缓存 7 天。 |

## 3. 前台公开数据缓存

| 数据 | 是否可缓存 | 当前 TTL/策略 | 失效要求 |
| --- | --- | --- | --- |
| 首页 bootstrap | 可以 | PWA `NetworkFirst`，最多 60 秒 | 后台改首页配置后，最多 1 分钟应看到新数据。 |
| 站点信息、首页运营配置 | 可以 | PWA `StaleWhileRevalidate`，最多 10 分钟 | 后台改站点信息、首页配置后，需要前端 invalidate 后台查询；前台最多等 TTL 自动刷新。 |
| 分类 | 可以 | PWA 最多 30 分钟，服务端内存 60 秒 | 后台改分类后，服务端清 `catalog` 内存缓存；前台可能受 PWA TTL 影响。 |
| 商品列表、商品标签 | 可以 | PWA 最多 5 分钟，服务端内存 60 秒 | 商品新增、编辑、上下架、批量改状态后，必须清 `catalog` 内存缓存。 |
| 商品详情、相关商品 | 可以 | PWA 最多 10 分钟 | 商品详情修改后，前台可能在 TTL 内短暂看到旧内容。 |
| 轮播图 | 可以 | PWA `NetworkFirst`，最多 60 秒，服务端内存 60 秒 | 后台改轮播图后，服务端清 `catalog` 内存缓存。 |
| 内容页 | 可以 | PWA 最多 24 小时 | 内容页有较长缓存，发布重要内容时建议同步清浏览器/PWA 或降低 TTL。 |
| 图片 | 可以 | PWA 最多 30 天，服务器 `/uploads` 7 天 | 上传文件名应带 hash；替换图片不要复用旧文件名。 |

## 4. 后台数据缓存

1. 后台接口默认不走 PWA 公共缓存。
2. 后台页面使用 React Query 缓存时，新增、编辑、删除、发布、下架、批量操作后必须 `invalidateQueries`。
3. 库存、订单、支付、监控、事件中心等变化快的数据，`staleTime` 应控制在 0-90 秒内。
4. 报表类数据可以短缓存，但导出和重新计算类接口必须走实时请求，并受高成本接口限流保护。

## 5. 修改数据后的失效动作

| 后台操作 | 必须失效 |
| --- | --- |
| 商品新增/编辑/上下架/删除/导入/批量状态 | 商品列表、商品详情、商品标签、库存相关后台查询、服务端 `catalog` 内存缓存。 |
| 分类新增/编辑/排序/删除 | 分类列表、商品列表、服务端 `catalog` 内存缓存。 |
| 轮播图新增/编辑/删除 | 轮播图、首页 bootstrap、服务端 `catalog` 内存缓存。 |
| 首页运营配置、导航、模块配置 | 首页 bootstrap、home-ops、后台 home-ops 查询。 |
| 站点配置、Logo、客服入口 | site-info、manifest/PWA 品牌相关接口、后台 site settings 查询。 |
| 内容页发布/编辑 | 对应内容页、后台内容列表。 |
| 优惠券、活动、营销配置 | 优惠券列表、营销看板、商品列表或首页活动区。 |
| 评论审核、隐藏、删除 | 评论列表、商品详情评论统计。 |
| 库存调整、采购、拆包/组装 | 库存列表、商品列表、商品详情、后台库存看板。 |

## 6. 新增缓存时的验收标准

1. 有明确 TTL。
2. 有明确 key 命名。
3. 有明确清理函数或 invalidate 位置。
4. 私密数据不能进入公共缓存。
5. 后台保存后，刷新前台页面能在预期时间内看到新数据。
6. 缓存服务 Redis 不可用时，接口不能直接崩溃，应降级为查数据库。
