# 数据中心统计合同

数据中心是订单、支付、退款、优惠券、库存和成本模块的结果层。任何无法从真实数据统计的字段，都不得用写死 0 冒充真实指标；应隐藏字段并返回 warnings 降级提醒。

## 核心口径

- 已支付订单：`payment_status` 属于 `paid` 或 `partially_refunded` 的订单。
- 有效订单：已支付且 `status` 不是 `cancelled` 的订单，用于商品、分类、活动、库存动销等销售归因。
- 有效销售额：已支付订单的 `total_amount` 汇总。
- 净销售额：已支付订单 `total_amount - refunded_amount`，旧库缺少 `refunded_amount` 时按未扣退款降级。
- 退款金额：`orders.refunded_amount` 汇总；缺字段时只能明确降级，不能伪造退款统计。
- 优惠金额：已支付订单的 `discount_amount` 汇总。
- 商品成本：优先使用订单或订单明细成本快照字段；缺成本订单必须暴露 `missing_cost_order_count` / `missing_cost_item_count`。
- 商品毛利：优先使用订单或订单明细毛利快照字段。
- 净利润：商品毛利 + 运费收入 - 物流成本 - 支付手续费 - 退款金额 - 经营支出。

## 时间字段

- 销售日报、销售月报：按订单 `created_at` 折算到马来西亚时区统计。
- 利润日报、利润月报：订单部分按 `created_at` 折算到马来西亚时区，经营支出按 `expense_date` 计入对应日/月。
- 商品分析、分类分析、订单分析、客户分析、活动分析、优惠券分析：默认按订单或业务记录创建/使用时间折算到马来西亚时区。
- 库存分析：近 7 天和近 30 天销量来自有效订单的 `order_items.qty`。

## 变更影响面

- 订单状态改动会影响销售、利润、商品、分类、订单、客户、活动、优惠券、库存报表。
- 支付状态改动会影响所有已支付订单口径相关报表。
- 退款逻辑改动会影响销售净额、退款金额、利润、订单退款率、优惠券净销售额。
- 优惠券字段改动会影响优惠券领取、使用、过期、优惠金额、带动销售额和 ROI。
- 商品成本字段改动会影响利润、商品毛利、分类毛利、活动毛利、优惠券毛利和缺成本提醒。
- 库存销量来自订单明细，不来自写死字段或当前库存倒推。

## 必跑检查

修改订单、支付、退款、优惠券、库存或成本模块后，必须执行：

```bash
npm run test:reports
npm run test:report-contract
npm run test:report-export
```

修改前端或后端报表 registry 后，必须执行：

```bash
npm run check:report-registry
```
