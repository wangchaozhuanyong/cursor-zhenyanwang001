## 二、分类说明

### A. 必须迁移 / 已解除的（运行时依赖）

- 站点代码、静态资源、Nginx `root`、`PROJECT_DIR`、CI `DEPLOY_PROJECT_DIR`：**不得再唯一绑定** `/www/wwwroot`。
- 部署脚本默认路径：**已**改为 `/var/www/click-send-shop`（见 `deploy/production-deploy.sh`、`deploy/ci-deploy.sh`、`deploy/deploy-wwwroot.sh` 等）。
- 验证脚本中**读取宝塔面板端口文件**的逻辑：**已**从 `verify-post-deploy.sh` 移除。

### B. 可以保留但建议迁移的

- 历史迁移/诊断文档可保留，方便排查曾经从宝塔迁出的服务器。
- 部署入口不再保留 `baota-*` 兼容脚本；新环境只使用：
  - `deploy/production-deploy.sh`
  - `deploy/ci-deploy.sh`
  - `deploy/verify-post-deploy.sh`
  - `deploy/verify-pm2.sh`

### C. 可以直接删除的（在确认无外部引用后）

- 服务器上：宝塔软件本身、`/www/server/panel`、`bt` 命令、宝塔自带 Nginx（若已切换系统 Nginx）。
- 仓库内：已确认不使用宝塔后，旧包装脚本已删除。

## 三、服务器待查清单（需在 SSH 上执行，无法在本仓库内自动完成）

| 项 | 检查命令 / 方法 | 若依赖宝塔则 |
|----|-----------------|-------------|
| 项目目录是否在 `/www/wwwroot` | `pwd` / `ls -la /www/wwwroot` | 按 `scripts/migrate-from-baota-path.sh` 迁到 `/var/www/...` |
| Nginx 是否宝塔生成 | `nginx -V 2>&1`；`which nginx`；是否 `/www/server/nginx` | 改用 `apt install nginx`，配置放 `/etc/nginx/sites-available/` |
| SSL 是否宝塔续签 | 证书路径是否在 `/www/server/panel/...` | 改用 `certbot` 或 `acme.sh` + systemd timer / cron |
| Node 是否面板启动 | `pm2 list` 来源；是否有「宝塔 Node 项目」 | 仅用 `pm2` + `ecosystem.config.cjs` 或 systemd |
| MySQL 是否仅宝塔面板管理 | `systemctl status mysql`；能否 `mysql` CLI 直连 | 保留 mysqld 服务即可；**禁止**依赖面板创建库用户（改用手动 SQL） |
| 计划任务 | `crontab -l`；宝塔「计划任务」里的脚本 | 改为 `crontab` 或 systemd timer |
| 防火墙 | `ufw status` / `iptables` vs 宝塔「安全」 | 改用 `ufw`/云安全组 |
| 宝塔进程 | `systemctl list-units | grep -i bt`；`which bt` | 迁移完成并验收后卸载（见 `03-删除执行清单.md`） |

## 四、结论（代码库）

- **业务代码**（`server/src`、`click-send-shop-main`）**无**宝塔 API 或面板路径硬依赖。
- **部署层**已通过 `production-deploy.sh` + 标准路径 + `deploy/nginx/site.prod.example.conf` + `deploy/systemd/README.md` 实现**可脱离宝塔**的默认形态；遗留 `baota-*` 兼容脚本已删除。
