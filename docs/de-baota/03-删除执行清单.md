# 《删除执行清单》（仅在迁移与验收全部通过后执行）

> **禁止**在网站未在新路径、新 Nginx、新 SSL、新进程守护下验证通过前执行本节。

## 前置条件（必须全部满足）

1. 项目已存在于 `/var/www/click-send-shop`（或你选定的 `/opt/...`），且 `bash deploy/production-deploy.sh` + `bash deploy/verify-pm2.sh` 均成功。
2. 系统 Nginx 已接管 80/443，域名与证书正常，`curl -I https://你的域名` 正确。
3. `pm2 list` 中 `gc-api` 为 `online`，且 `pm2 startup` 已配置，**重启服务器**后进程自动恢复。
4. 数据库：应用仍能通过 `server/.env` 连接（与是否曾用宝塔安装 MySQL **无关**，只要 `mysqld` 服务正常）。
5. 已备份：数据库全量 dump、旧站点目录 tarball（可选但强烈建议）。

## 删除步骤（顺序建议）

| 步骤 | 操作 | 风险 |
|------|------|------|
| 1 | 停止宝塔面板服务（名称因版本而异，示例）：`sudo bt stop` 或 `sudo systemctl stop bt` / `panel` | 确认 Nginx 已非宝塔进程占用 80/443 |
| 2 | 确认无进程监听宝塔端口（如 888）：`ss -ltnp` | 低 |
| 3 | 使用官方卸载脚本（**以宝塔文档为准**）：通常在面板或官网提供的 `bt uninstall` / 卸载脚本 | **高**：误删数据；务必已备份 |
| 4 | 手动清理残留（谨慎）：`/www/server/panel`、`/www/server/nginx`（若已不用）、`bt` 命令 | 勿删 `/var/www/click-send-shop` |
| 5 | `sudo systemctl daemon-reload`，确认无 `bt` 相关 failed 单元 | 低 |
| 6 | 再次验证站点与 API、`verify-pm2.sh` | 阻断项 |

## 回滚

- 若卸载后发现问题：**恢复数据库备份** + 从 tarball 还原旧目录（或重新 `git clone`）+ 临时恢复旧 Nginx 配置（应提前保留副本）。
