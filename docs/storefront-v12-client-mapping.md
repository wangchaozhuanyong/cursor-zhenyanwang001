# 客户端 V12 页面映射与落地边界

本清单基于 V12 设计稿，只用于客户端前台重构拆分。后台、支付真实联调、生产部署不包含在本轮。

## 落地原则

- 客户端默认中文；前台多语言入口默认关闭。
- 商品价格、活动资格、库存、订单金额、支付状态都以后端返回为准。
- V12 静态 HTML 只作为信息架构和视觉方向，不直接复制硬编码样式、假图片、假数据。
- 旧 `/deals` 链路继续兼容跳转到 `/promotions`。

## 不能照抄的部分

- 设计稿里的 Billplz / FPX 演示不代表当前真实支付已开通。
- 设计稿里的余额、礼品卡、活动库存、支付结果是假数据，前台只能展示后端或功能开关返回的真实状态。
- 设计稿里的 EN / MS 切换不落到客户端；后台可保留中英文。
- 设计稿里的后台表格和规则编辑不进入本轮客户端重构。
