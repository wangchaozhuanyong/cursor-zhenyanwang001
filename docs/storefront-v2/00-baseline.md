## 当前问题

- 首页模块过多，首屏重点不够集中。
- 商品卡片承载活动、热销、新品、标签、价格、库存等多类信息，视觉和高度容易不稳定。
- 活动展示分散：秒杀、优惠券、满减、促销 Banner 各自请求和展示。
- 全局挂载组件较多，Cookie、PWA、Tracking、Toast、兼容提示等不应影响购物首屏和结算路径。
- 现有 `StoreCardOverlapFix` 依赖全局 `!important` 修补移动端卡片布局，不能作为长期方案。
- 活动已存在于商品详情、购物车、结算页，但承接体验不完整，缺少活动面板、凑单提示和结算解释。
- `/api/home/marketing` 当前缓存策略对价格、券、库存类活动信息偏长。

## 不允许破坏

- 登录注册。
- 购物车。
- 结算。
- 订单。
- 支付。
- 优惠券领取和使用。
- 积分。
- 后台权限。
- 现有 API 路径兼容。
- 订单金额、库存扣减、优惠券核销、积分抵扣等最终业务规则必须以后端为准。

## 本分支目标

- 重构前台购物体验。
- 建立统一活动展示模型。
- 优化首页和商品卡。
- 补齐活动承接链路。
- 降低首屏复杂度。
- 保留旧组件作为回退，避免一次性破坏现有链路。
