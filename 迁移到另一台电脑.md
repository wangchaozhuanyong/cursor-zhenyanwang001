# 把源码 + 本机 MySQL 迁到另一台电脑

你在**当前这台 Windows 电脑**上按下面做即可。完成后会得到一个 **ZIP 压缩包**，拷到 U 盘/网盘/微信文件传输助手，在另一台电脑解压后继续部署。

---

## 开始前（只需确认 2 件事）

1. **本机 MySQL 已启动**，项目能正常连上库（你平时能跑起来就行）。
2. **已配置 `server/.env`**，至少包含这些键（没有就复制 `server/.env.example` 为 `server/.env` 再填）：
   - `DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`（没有密码可留空）
   - `DB_NAME`（默认多为 `click_send_shop`）

脚本会按 `server/.env` 自动导出数据库，**你不需要自己记用户名和库名**。

---

## 方式 A：一键打包（推荐）

在 **PowerShell** 里执行（路径按你实际项目位置改）：

```powershell
cd "D:\CURSOR\购物网站"
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force
.\scripts\打包迁移到另一台电脑.ps1
```

执行结束后：

- 会在**桌面**生成：`shop-migrate-YYYYMMDD-HHmmss.zip`
- 压缩包里包含：项目源码（已去掉 `node_modules` 等大目录）+ `backups` 里刚导出的 `.sql` 文件 + 本说明

把 ZIP 拷到另一台电脑即可。

---

## 方式 B：只导出数据库（手动）

若你不想打 ZIP，只要 SQL 文件：

```powershell
cd "D:\CURSOR\购物网站"
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force
.\scripts\backup-mysql.ps1
```

完成后看终端里提示的路径，一般在 `backups\click_send_shop_时间戳.sql`。

---

## 另一台电脑上做什么

1. 解压 ZIP 到任意目录（例如 `D:\work\购物网站`）。
2. 安装 **Node.js 20+**、**Git**（若要从零装依赖）。
3. 在项目里复制环境变量：
   - 把 `server\.env` 检查一遍（尤其是 `DB_*`），部署到 AWS 时再改成 RDS 地址。
4. 安装依赖（在解压后的项目里）：
   ```powershell
   cd "解压路径\click-send-shop-main\click-send-shop-main"
   npm ci
   cd "解压路径\server"
   npm ci
   ```
5. **导入 MySQL**（在已安装 MySQL 的机器上，或以后导入 RDS）：
   ```bash
   mysql -u root -p < backups\你的备份文件.sql
   ```
   若目标库名不同，先用客户端建库再导入。

---

## 说明

- **我不能替你在云端执行**：数据库导出必须在**你本机**跑 `mysqldump`（已通过 `backup-mysql.ps1` 封装）。
- ZIP 里**不包含** `node_modules`，体积会小很多；另一台电脑需要再执行 `npm ci`。
- `backups/` 目录在 Git 里被忽略，但一键脚本会把新生成的 `.sql` 打进 ZIP。

---

## 若备份脚本报错

- **找不到 mysqldump**：安装 MySQL 客户端，或把 `mysqldump.exe` 所在目录加入系统 PATH。脚本也会尝试使用  
  `C:\Program Files\MySQL\MySQL Server 8.0\bin\mysqldump.exe`。
- **Access denied**：检查 `server/.env` 里 `DB_USER`、`DB_PASSWORD` 是否与本机 MySQL 一致。
- **没有 `server/.env`**：复制 `server/.env.example` 为 `server/.env`，填好本机数据库再运行脚本。
